AP Computer Science Principles · Number Systems

Binary, Decimal & Hexadecimal

Every value inside a computer is just on/off switches. This page shows how the same number can wear three different outfits — and how to change its clothes by hand.

same number — three different outfits! ✓

01 · THE BIG IDEA

A "number system" is just a set of symbols + a place-value rule

We count in base 10 because we have ten fingers. Computers only have two states — on and off — so they count in base 2. Hexadecimal (base 16) is the shorthand programmers use so they don't go cross-eyed reading long strings of bits.

Throughout this page we'll follow one value through all three systems as our running example:

182  =  1011 0110  =  B6
# decimal       binary           hex

Same number, three costumes. Once you can move between them, hexadecimal stops looking scary.

02 · THE THREE SYSTEMS

Meet the cast

The number of distinct symbols a system uses is its base. That's the only thing that really changes between them.

SystemBaseDigits usedWhere you meet it
Decimal100 1 2 3 4 5 6 7 8 9Everyday math
Binary20 1How all data is stored (on/off)
Hex160–9 then A B C D E FColors, memory addresses
Vocabulary to memorize. One binary digit is a bit. Four bits is a nibble — and one nibble is exactly one hex digit. Eight bits is a byte — exactly two hex digits, holding values 0–255.

1 nibble = 4 bits = 1 hex digit

03 · PLACE VALUE

The one idea behind all of it

Every system works the same way: each position is worth baseposition, counting from 0 on the right. Multiply each digit by its position value, then add.

# decimal — powers of 10
182
= 1×10² + 8×10¹ + 2×10⁰
= 100 + 80 + 2
= 182
# binary — powers of 2
10110110
= 1×2⁷ + 1×2⁵ + 1×2⁴ + 1×2² + 1×2¹
= 128 + 32 + 16 + 4 + 2
= 182

If you understand the decimal box, you already understand binary and hex. Only the base changes.

05 · DECIMAL → BINARY

Divide by two, collect the remainders

Repeatedly divide by 2 and write down each remainder. The remainders, read bottom to top, are your binary digits.

# convert 182 to binary
182 ÷ 2 = 91  remainder 0
 91 ÷ 2 = 45  remainder 1
 45 ÷ 2 = 22  remainder 1
 22 ÷ 2 = 11  remainder 0
 11 ÷ 2 =  5  remainder 1
  5 ÷ 2 =  2  remainder 1
  2 ÷ 2 =  1  remainder 0
  1 ÷ 2 =  0  remainder 1
# read upward ↑      → 1011 0110
Shortcut: subtract the biggest power of two. Largest power that fits in 182 is 128 → write a 1, leaves 54. 64 is too big → 0. 32 fits → 1, leaves 22 … building 10110110 left to right. Use whichever method clicks for you.
06 · HEX → BINARY

The easiest conversion there is

Here's the magic fact that makes hex and binary best friends: 16 = 2⁴. That means every single hex digit is worth exactly 4 bits.

★ The mini sub-problem trick. Never convert a long hex number all at once. Instead, treat each hex digit as its own tiny 4-bit puzzle, solve each separately, then snap the nibbles together like LEGO. A 6-digit hex number becomes six trivial 4-bit problems.

The only table you need — each hex digit → 4 bits

# convert B6 — one mini-problem per digit
B = 11 = 1011
6 =  6 = 0110
# snap together →  1011 0110

16 = 2⁴ — that's the whole trick!

07 · BINARY → HEX

Same trick, run backwards

Slice the bits into groups of 4, starting from the right. Pad the leftmost group with zeros if needed. Convert each nibble to a single hex digit.

# convert 10110110 — group into 4s from the right
1011  0110
1011 = 11 = B
0110 =  6 = 6
# read left to right →  B6

Padding example: 110110 has 6 bits → pad to 0011 0110 → 36.

Why this matters. Hex is just a shorthand for binary that humans can read. Two hex digits compress eight bits into something you can say out loud — which is why programmers write colors and memory addresses in hex, not raw binary.
08 · COLORS, THREE WAYS

The same color in binary, decimal, and hex

09 · HOW MANY COMBINATIONS?

With n positions, how many values can you make?

The rule is delightfully simple:

number of values = base^digits

A 24-bit color has 24 binary positions → 2²⁴ possible values. But 24 bits is also 6 hex digits, and also 3 bytes. So the same count can be written three ways:

2²⁴  =  16⁶  =  256³  =  16,777,216
# because 16 = 2⁴, so 16⁶ = (2⁴)⁶ = 2²⁴
# and    256 = 2⁸, so 256³ = (2⁸)³ = 2²⁴

Grouping bits never changes how many values exist — it only changes how you write them down.

Calculator: pick any base and number of digits

›2^8 = 256 possible values
An 8-bit value (one byte) → 0 to 255.
10 · OTHER BASES EXIST TOO

A quick tour past the big three

Binary, decimal, and hex are the headliners, but any whole number ≥ 2 can be a base — you just need that many distinct symbols. A few you might bump into:

BaseNameWhat's interesting
8OctalDigits 0–7. Since 8 = 2³, each digit is 3 bits. You've seen it in Unix file permissions like chmod 755.
64Base64Uses 64 symbols (A–Z, a–z, 0–9, + /) to pack binary data into plain text — email attachments, data URLs. More an encoding than arithmetic.
5Base 5Digits 0–4. Works perfectly, just rarely useful. Proof the rule is general.
83Base 83Real systems use it (BlurHash, which encodes blurry image previews). You'd need 83 distinct symbols for a single digit.

Why does computing love 2, 8, and 16 specifically? Because 8 and 16 are powers of 2, so they group bits cleanly with nothing lost. Base 10 is the odd one out — handy for humans, awkward for hardware.
