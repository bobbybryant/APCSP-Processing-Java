01 — Foundations

What is a data type?

When you store information in a Processing program, the computer needs to know what kind of information it is. Is it a whole number? A decimal? A piece of text? A yes/no answer? That "kind" is called a data type.

The data type tells the computer two important things: what values are allowed (you can't store the word "cat" in a number), and how much memory to set aside. Memory is measured in bytes — one byte is 8 bits, and a bit is a single 0 or 1.

Why memory size matters Bigger types can hold bigger values but take up more space. On a tiny sketch it doesn't matter, but the habit of picking the right-sized type is a real computer-science skill. A byte can only count to 255 — an int can count past 4 billion.
02 — Reference

The nine data types

These are the types we learned. The ones marked with a ★ are the four we reach for most of the time. "Bytes" is how much memory one value takes up.

TypeBytesHoldsRange / NotesUse it for…
byte1Whole numbers0 to 255Rarely used; tiny whole numbers
short2Whole numbers0 to 65,535Rarely used
int ★4Whole numbers0 to ≈ 4.3 billionDefault whole number — counters, coordinates, scores
long8Whole numbers0 to an astronomically large numberWhen an int isn't big enough
float ★4Decimal numbers≈ 7 digits of precisionDefault decimal — prices, speeds, percentages
double8Decimal numbers≈ 15 digits of precisionWhen you need very precise decimals
char1*A single characterOne symbol in single quotes: 'A'One letter, digit, or symbol
String ★varies**TextAny text in double quotes: "hello"Default for text — names, words, sentences
boolean ★1 bittrue / falseOnly two values: true or falseYes/no answers — isGameOver, hasKey
Things to remember in our class Ranges are unsigned. We treat whole-number types as unsigned, so they start at 0 (a byte is 0–255, not −128 to 127).
*char = 1 byte for us. In the real world a char is 2 bytes (it can hold any Unicode symbol), but in our class we count it as 1 byte.
**String is special: it's not a basic ("primitive") type — it's an object built from many characters, so its size depends on how long the text is, and it's the only type that starts with a capital letter.
boolean: only needs 1 bit of information (true = 1, false = 0), but computers don't address individual bits — they work in bytes. So a boolean is stored as a full 1-byte container even though only one bit inside it is used.
Quotes matter: char uses single quotes; String uses double quotes — mixing these up is the #1 beginner error.
03 — The shortlist

The four we prefer

You could memorize all nine, but in practice almost everything you write uses just these four. When you're deciding what type to use, start here and only reach for something else if you have a reason.

int

  • Whole numbers
  • Coordinates, counts, scores
  • int lives = 3;

float

  • Decimal numbers
  • Speed, money, ratios
  • float speed = 2.5;

String

  • Text in "double quotes"
  • Names, labels, words
  • String name = "Ava";

boolean

  • true or false only
  • On/off, yes/no states
  • boolean win = false;
Quick decision rule Whole number? int. Has a decimal point? float. Text? String. Just yes-or-no? boolean.
04 — Variables

What variables are (and why we use them)

A variable is a named container that holds a value. Instead of writing the number 500 all over your sketch, you store it once in a variable and use the variable's name everywhere else.

Three big reasons we use variables:

1. Change it once, update everywhere

If twelve shapes are positioned using houseX and you want to move the whole house, you change one line instead of twelve numbers.

2. Names explain your code

A reader sees playerSpeed and instantly knows what it means. A bare 2.5 floating in your code explains nothing.

3. Relationships stay connected

When you build shapes relative to a variable (the roof sits at houseX + 50), they all move together automatically. That's exactly what we did with the house — try the live demo in section 07.

05 — The rules

Rules for naming variables

Processing is picky about variable names. Some rules are required (break them and your sketch won't run), and some are conventions (good habits that make your code readable).

Required rules

  • Must start with a letter. Not a number, not a symbol.
  • No special characters except the underscore ( _ ). No spaces, dashes, dollar signs, dots, etc.
  • After the first letter, you may use letters, numbers, and underscores.
  • No spaces. high score is two words and will error.
  • Case-sensitive. houseX and housex are two different variables.
  • Can't be a reserved name — any word Processing already uses. This includes type keywords like int and float, AND built-in names like width, height, mouseX, mouseY, color, and size.

Conventions (strongly recommended)

  • Use camelCase: start lowercase, capitalize each new word — houseX, playerSpeed, isGameOver.
  • Be descriptive. score beats s. Future-you will thank present-you.

    06 — Syntax

    Declaration, initialization & assignment

    These three words sound similar but mean different things. Knowing the difference is a classic exam question, so let's nail it down.

    1. Declaration — create the container

    Declaring means telling the computer the variable's type and name. This sets aside the memory, but it doesn't put a value in yet.

    2. Initialization — give it its first value

    Initializing means giving a variable its very first value. You can do it on its own line, or combine it with the declaration (this is what we usually do).

    3. Assignment — change the value later

    Assigning means giving the variable a new value at any point after it already exists. You do not write the type again — it's already declared.

    The one-line summary Declare = make it (type + name). Initialize = first value. Assign = any value after that. The type is written only when you declare — never again.

    09 — Self-check

    Thought questions

    Try each one on your own first, then tap Reveal answer to check yourself. No peeking until you've committed to an answer!

    1
    Choose the best data type for each: (a) a person's age, (b) the price of a coffee, (c) a username, (d) whether the lights are on.
    Answer(a) int — age is a whole number. (b) float — prices have decimals like 3.50. (c) String — text. (d) boolean — it's either on (true) or off (false).
    2
    Which of these are valid variable names, and why are the others invalid?
    2myVar   my var   score_1   mouseX   playerName   total$   _count
    AnswerValid: score_1, playerName, and technically _count (underscore start is allowed, but we prefer starting with a letter).
    Invalid: 2myVar (starts with a number), my var (has a space), mouseX (reserved — it's a built-in name Processing already uses), total$ (the $ is a special character we don't allow). Use the checker in section 05 to confirm!
    3
    In our class a byte holds values from 0 to 255. What kind of problem happens if you try to store 300 in a byte, and which type should you use instead?
    Answer300 is out of range for a byte (the max is 255), so it won't fit / will "overflow." Use an int — it easily holds 300 and goes up past 4 billion.
    4
    Given int houseX = 500; and int houseY = 300;, write the three points of a roof triangle() so the roof sits on top of a 120-wide wall, with the peak 70 pixels above the wall and centered.
    Answer
    The peak's x is houseX + 60 (half of 120 = centered), and its y is houseY - 70 because smaller y is higher on the screen.
    5
    How many total bytes of memory do these three variables use together?
    int a;   float b;   char c;
    Answer9 bytes. int = 4, float = 4, char = 1. So 4 + 4 + 1 = 9.
    6
    Your sketch uses the number 450 in eight different places to position a tree. Your teacher asks you to move the tree. Explain why a variable would have saved you time — and write the line that declares and initializes it.
    AnswerWith a variable you'd change one line instead of hunting down all eight 450s (and risking missing one). Declare and initialize it like: int treeX = 450; — then use treeX everywhere. To move the tree, you edit only that single line.
    10 — At a glance

    Cheat sheet

    The four favorites

    • int — whole numbers
    • float — decimals
    • String — "text"
    • boolean — true/false

    Memory sizes (bytes)

    • byte 1 · short 2
    • int 4 · long 8
    • float 4 · double 8
    • char 1 · String varies

    Naming: required

    • Start with a letter
    • Only letters, numbers, _
    • No spaces, case-sensitive
    • Not a reserved/built-in name

    Naming: style

    • camelCase: playerSpeed
    • Be descriptive
    • Declare type once
    • Reassign with no type

    Quotes matter

    • char → 'single'
    • String → "double"
    • String starts Capital

    Screen reminder

    • x grows → right
    • y grows → down
    • up = smaller y
