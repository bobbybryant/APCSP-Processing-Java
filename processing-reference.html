Processing reference — your toolbox

Every move you've learned in Processing, in one place: how to draw it, how to read the mouse and keyboard, how to store and search data. Grab what you need. These are the tools — how you combine them into your Create Task is up to you.

01

Sketch skeleton

Almost every sketch is built from two functions. setup() runs once at the start; draw() runs over and over, about 60 times a second — that loop is what makes things move.

void setup() {
  size(600, 400);     // window is 600 wide, 400 tall -- runs once
}

void draw() {
  background(255);    // wipe the screen -- runs ~60x a second
  ellipse(mouseX, mouseY, 40, 40);
}

A still picture? Drop the loop.

If nothing needs to move, you can put everything in setup() and skip draw() entirely.

void setup() {
  size(400, 400);
  background(255);
  ellipse(200, 200, 80, 80);   // drawn once, never moves
}
background = an eraser Calling background() at the top of draw() wipes the screen each frame. Leave it out and your shapes smear and pile up — sometimes that's the look you want.
02

The coordinate grid

The screen is a grid of pixels. The top-left corner is (0, 0). X grows to the right; Y grows downward (not up, like in math class).

mouseX = 0 mouseY = 0

Move your mouse over the box. (0,0) is the top-left corner; X grows going right, Y grows going down.

So ellipse(width/2, height/2, 40, 40) always sits dead center, whatever your window size.

03

Drawing shapes

The everyday shapes. Notice ellipse measures from its center, while rect measures from its top-left corner by default.

ellipse(cx, cy, w, h);     // circle / oval (center x, center y)
rect(x, y, w, h);          // rectangle (top-left corner)
line(x1, y1, x2, y2);      // a straight line between two points
triangle(x1, y1, x2, y2, x3, y3);
point(x, y);               // a single dot
arc(x, y, w, h, start, stop);   // angles in radians

Switching how they're measured

rectMode(CENTER);     // now rect(x, y, ...) is centered, like ellipse
ellipseMode(CORNER);  // now ellipse uses a corner instead of the center
order matters Shapes stack like paper — whatever you draw last sits on top. Draw your background first, your characters after.
04

Color & style

Set a color before you draw a shape, and it sticks until you change it. One number is grayscale; three numbers are red / green / blue (0–255 each).

background(255);          // one number = grayscale (0 black, 255 white)
fill(255, 0, 0);          // three numbers = red, green, blue
stroke(0);                // outline color
strokeWeight(4);          // outline thickness
noStroke();               // no outline at all
noFill();                 // outline only, hollow inside
fill(0, 150, 255, 80);    // a 4th number = alpha (see-through)

Mix a color

Drag the sliders — this is exactly what the three numbers in fill() do.

R 232
G 114
B 90
fill(232, 114, 90)
05

Text on screen

Use text() to draw words or numbers. Glue text and variables together with +.

textSize(28);
textAlign(CENTER, CENTER);
fill(0);
text("Score: " + points, width/2, 40);   // glue words + a variable with +
style first, then draw Set textSize, textAlign, and fill before the text() call, just like with shapes.
06

Variables you get for free

Processing keeps these filled in for you every frame — you never declare them, just read them.

width, height        // the size of your window
mouseX, mouseY       // where the mouse is right now
pmouseX, pmouseY     // where the mouse was the frame before
mousePressed         // true while a mouse button is held
keyPressed           // true while any key is held
key                  // the most recent character key ('a', ' ', etc.)
keyCode              // code for special keys (UP, DOWN, LEFT, RIGHT...)
frameCount           // how many frames have run since the start
use width & height, not numbers Writing width instead of typing 600 means your sketch still lines up if you ever change the window size.
07

Random numbers

random() hands back a decimal. Wrap it in int() to turn it into a whole number.

float f = random(10);        // a decimal from 0 up to (but not including?) 10
int n = int(random(6));      // wrap it in int() to chop off the decimal
int die = int(random(1, 7)); // give two numbers for a range

Roll it

–
find the pattern yourself Roll it a few hundred times. What is the highest number you ever see? The lowest? That tells you exactly which numbers int(random(1, 7)) is allowed to land on — figure it out from the tally instead of guessing.
08

Handy math

A handful of helpers that come up constantly.

dist(x1, y1, x2, y2);   // straight-line distance between two points
map(v, 0, 100, 0, 255); // re-scale v from the 0-100 range onto 0-255
constrain(v, 0, 255);   // clamp v so it never leaves 0..255
abs(-7);                // 7  (distance from zero)
min(a, b);  max(a, b);  // smaller / larger of two values
10 % 3;                 // 1  (the remainder -- great for wrapping & evens/odds)
the % operator is sneaky-useful i % 2 == 0 is true for even numbers. And anything % n stays in the range 0 to n−1 — perfect for looping a value back around.
09

Making decisions

An if runs its block only when the test is true. Chain else if / else to handle several cases.

if (score >= 90) {
  fill(0, 200, 0);
} else if (score >= 70) {
  fill(255, 200, 0);
} else {
  fill(200, 0, 0);
}

The comparison & logic operators

==   equal to            !=   not equal to
>    greater than        <    less than
>=   at least            <=   at most
&&   AND (both true)     ||   OR (either true)
!    NOT (flips it)
one = vs two == = stores a value (x = 5). == checks if two things are equal (x == 5). Mixing them up is the most common bug there is.
10

Repeating: loops

A for loop repeats a known number of times. Start the counter at 0, go up by 1, and put the positioning math inside the body.

for (int i = 0; i < 5; i++) {
  ellipse(40 + i * 60, 80, 30, 30);   // start at 0, +1 each time
}

while — repeat until a condition changes

int x = 0;
while (x < width) {
  line(x, 0, x, height);
  x = x + 40;
}

A loop inside a loop = a grid

for (int row = 0; row < 4; row++) {
  for (int col = 0; col < 6; col++) {
    rect(col * 40, row * 40, 36, 36);   // a whole grid
  }
}
11

Your own functions

A function bundles steps under a name so you can reuse them. Values in the parentheses are parameters — inputs the function reacts to.

void drawStar(int x, int y, int size) {   // 3 parameters
  // ...draw a star using x, y, size...
}

drawStar(100, 100, 30);   // call it -- now it runs

Handing a value back

Swap void for a type (like int) and use return to send an answer back.

int addOne(int n) {
  return n + 1;      // hands a value back to whoever called it
}

int result = addOne(5);   // result is now 6
one job each A good function does exactly one thing. If a function is drawing a tree and also keeping score, that's really two functions wearing a trench coat — split them.
12

Arrays & lists

When you need to hold many values under one name, you have two choices.

Array — fixed size

int[] scores = {88, 95, 72, 60, 91};   // fixed size, set up front
scores[0];          // read the first item -> 88
scores[2] = 100;    // change an item
scores.length;      // how many items -> 5

Or make empty slots to fill later:

int[] data = new int[10];   // 10 empty slots (start as 0)
data[3] = 42;               // fill a slot by its index

ArrayList — grows and shrinks

ArrayList<Integer> nums = new ArrayList<Integer>();
nums.add(88);       // grows as you add
nums.get(0);        // read the first item -> 88
nums.set(0, 100);   // change an item
nums.size();        // how many items -> 1
nums.remove(0);     // take one out

Visiting every item

Loop from 0 up to the size to touch each one:

for (int i = 0; i < scores.length; i++) {
  println(scores[i]);   // visit every item, one at a time
}
two cheat words Array → .length and scores[i]. ArrayList → .size() and scores.get(i). Both start at index 0.

The code behind each button

Does it exist? — flip a flag to true the moment you find a match.

boolean found = false;
for (int i = 0; i < list.length; i++) {
  if (list[i] == 7) {
    found = true;
  }
}

How many times? — add one to a counter on every match.

int count = 0;
for (int i = 0; i < list.length; i++) {
  if (list[i] == 7) {
    count = count + 1;
  }
}

How many meet a criteria? — same idea, but the test is a range.

int passing = 0;
for (int i = 0; i < list.length; i++) {
  if (list[i] >= 5) {
    passing = passing + 1;
  }
}
14

Mouse

There are two ways to deal with the mouse, and the difference matters.

The mousePressed variable — held down?

Check it inside draw(). It's true the entire time a button is down.

void draw() {
  background(255);
  if (mousePressed) {       // true the whole time a button is down
    fill(255, 0, 0);
  } else {
    fill(200);
  }
  ellipse(mouseX, mouseY, 50, 50);
}

The mousePressed() function — the moment of a click

This runs once, right when you click. Great for counting clicks or toggling things.

void mousePressed() {   // runs ONCE the moment you click
  // ...react to the click...
}

void mouseReleased() { }   // when you let go
void mouseDragged()  { }   // while moving with a button held

Is the mouse over a shape?

Measure the distance from the mouse to the shape's center.

// is the mouse inside a circle at (cx, cy) with radius r?
boolean overCircle(float cx, float cy, float r) {
  if (dist(mouseX, mouseY, cx, cy) < r) {
    return true;
  } else {
    return false;
  }
}
dist to center = 0 mouse is outside

The circle turns teal when dist(mouseX, mouseY, cx, cy) drops below its radius.

don't get them confused mousePressed (no parentheses) is a true/false you read. mousePressed() (with parentheses) is a function that fires once. Same name, very different behavior.
15

Keyboard & arrow keys

Same split as the mouse: a variable for "is it held," and a function for "the instant it was pressed."

Is a key held? (in draw)

void draw() {
  background(255);
  if (keyPressed && key == ' ') {   // is the spacebar held?
    fill(0);
  } else {
    fill(255, 180, 0);
  }
  rect(150, 150, 100, 100);
}

React once to a letter or the spacebar

void keyPressed() {     // runs once per key press
  if (key == 'r') {
    // ...do something when R is pressed...
  }
  if (key == ' ') {
    // ...spacebar...
  }
}

The arrow keys

Arrow keys aren't letters — each one has a number. Check keyCode against 37 (left), 38 (up), 39 (right), and 40 (down).

int x = 200;
int y = 200;
int speed = 5;

void keyPressed() {
  if (keyCode == 37) { x = x - speed; }   // left
  if (keyCode == 38) { y = y - speed; }   // up
  if (keyCode == 39) { x = x + speed; }   // right
  if (keyCode == 40) { y = y + speed; }   // down
}

Try it

x = 270 y = 130 last keyCode = –

Click the box first, then press the ↑ ↓ ← → keys to move the square.

want smooth gliding? keyPressed() fires in little bursts (like holding a key in a text box). For buttery movement, keep a true/false for each arrow and update the position inside draw() — a nice upgrade once the basic version works.
16

Useful recipes

Small, complete patterns you can adapt. They're ingredients, not your whole dish.

Slide across and wrap around

int x = 0;
int speed = 3;

void draw() {
  background(255);
  ellipse(x, 200, 40, 40);
  x = x + speed;          // nudge it every frame
  if (x > width) {
    x = 0;                // jump back to the left edge
  }
}

Bounce off the walls

int x = 100;
int speed = 4;

void draw() {
  background(255);
  ellipse(x, 200, 40, 40);
  x = x + speed;
  if (x > width || x < 0) {
    speed = -speed;       // flip direction at a wall
  }
}

Toggle something on a click

boolean on = false;

void mousePressed() {
  on = !on;               // flip true<->false each click
}

void draw() {
  if (on) {
    background(0);
  } else {
    background(255);
  }
}

Count clicks

int clicks = 0;

void mousePressed() {
  clicks = clicks + 1;
}

void draw() {
  background(255);
  fill(0);
  textSize(40);
  text(clicks, width/2, height/2);
}
17

Debugging

When something's wrong, println() is your best friend. It prints to the black console at the bottom of the Processing window so you can see what your program is actually doing.

println("got here");          // prints to the black box at the bottom
println("mouseX is " + mouseX);
println(score);               // peek at a variable's value
when you're stuck Drop a println inside the part that's misbehaving. If its message never shows up, that code is never running — which is usually the real problem (often a function you forgot to call).
