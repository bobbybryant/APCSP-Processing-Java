Arrays & Lists:
storing many things at once

You already know arrays. Today we add two flexible cousins — IntList and StringList — and figure out when each one is the right tool. The secret weapon for all three is the same: the loop.

Start here

Why do we even need arrays?

Say you're tracking quiz scores and you want to print which ones passed (70 or higher). With separate variables, you'd write a new variable and a new if for every single student:

int score1 = 85;
int score2 = 62;
int score3 = 90;
int score4 = 71;
int score5 = 55;

if (score1 >= 70) { println("Student 1 passed"); }
if (score2 >= 70) { println("Student 2 passed"); }
if (score3 >= 70) { println("Student 3 passed"); }
if (score4 >= 70) { println("Student 4 passed"); }
if (score5 >= 70) { println("Student 5 passed"); }

Five scores, ten lines of near-identical code. Now imagine your class has 30 students. Or 300. You'd be copy-pasting that if forever, changing one number each time. That's the pain arrays were invented to kill.

An array lets you store all the scores under one name, and a loop runs the check once — no matter how many scores there are:

int[] scores = {85, 62, 90, 71, 55};

for (int i = 0; i < scores.length; i++) {
  if (scores[i] >= 70) {
    println("Student " + (i + 1) + " passed");
  }
}
The whole point ↓ The array-and-loop version does not get longer when you add more data. Drag the slider below and watch.

Refresher

A 30-second array recap


You've been living in arrays, so this is just a tune-up. An array is a row of numbered boxes, all the same type, created with a fixed size.



int[] scores = {85, 62, 90, 71, 55};   // five boxes, filled
int[] empties = new int[5];            // five boxes, all 0



Every box has an index, and indexes always start at 0:






You reach into a box with square brackets:


scores[0]                   // reads 85  (the FIRST box)
scores[scores.length - 1]   // reads 55  (the LAST box)
scores[2] = 100;            // overwrites the third box with 100
scores.length               // 5  (how many boxes — no parentheses!)




  The last index is length − 1 ↓
  Counting starts at 0, so five boxes are numbered 0–4 — the last is always at length - 1, never length. Reach for scores[scores.length - 1] instead of hardcoding scores[4], and it keeps working even after the array changes size. Step one box too far — scores[5] or scores[scores.length] — and Processing crashes with ArrayIndexOutOfBoundsException. That off-by-one is the single most common array bug there is.





  Watch this one ↓
  An array's size is locked the moment you make it. int[5] is always five boxes — you can't append a sixth or delete one. That single limitation is exactly what IntList and StringList fix.


  
The big idea

Make it work for one. Then let the loop scale it.


Here's the mindset that makes arrays click: first solve the problem for a single item, then wrap that exact logic in a loop so it repeats for every item.



Checking one score is easy:


if (scores[0] >= 70) {        // just the first box
  println("Passed!");
}



That's the whole job — for one box. A loop takes that same body and runs it on box 0, then box 1, then box 2… changing only the index i each time. You write the logic once; the loop does it five times (or five hundred).



for (int i = 0; i < scores.length; i++) {
  if (scores[i] >= 70) {              // the SAME check, now on box i
    println("Student " + (i + 1) + " passed");
  }
}

New tool

Meet IntList: an array that can grow


Arrays are great when you know the size up front. But what if items get added and removed while the program runs — a high-score list, a shopping cart, players joining a game? That's the job for an IntList. It holds int values like an array, but it can stretch and shrink.



IntList scores = new IntList();   // starts EMPTY (size 0)
scores.append(85);                // size 1  →  [85]
scores.append(62);                // size 2  →  [85, 62]
scores.append(90);                // size 3  →  [85, 62, 90]



Notice you never declared a size. You just keep append-ing and it keeps growing. Here are the moves you'll use most:




  

    
      
append(v)Add value v to the end of the list.

      
get(i)Read the value at index i.

      
set(i, v)Overwrite index i with value v.

      
size()How many values are in the list right now (parentheses!).

      
remove(i)Delete index i; everything after it shifts left.

      
hasValue(v)Returns true/false — is v in the list?

      
index(v)Returns the index where v first appears (or -1).

The loop barely changes


Walking an IntList uses the same loop shape as an array. Only two things swap: .length becomes .size(), and [i] becomes .get(i).


for (int i = 0; i < scores.size(); i++) {   // .size() not .length
  if (scores.get(i) >= 70) {                // .get(i) not [i]
    println("Student " + (i + 1) + " passed");
  }
}

  
New tool

Meet StringList: the same idea, for words


A StringList is IntList's twin — it grows and shrinks the same way and uses the same method names — but it holds String values instead of numbers. Names, words, answers, file names: anything text.



StringList names = new StringList();   // empty
names.append("Sam");                   // ["Sam"]
names.append("Maria");                 // ["Sam", "Maria"]
names.append("Devin");                 // ["Sam", "Maria", "Devin"]

names.get(1)            // "Maria"
names.set(0, "Sammy");  // ["Sammy", "Maria", "Devin"]
names.size()            // 3
names.hasValue("Devin") // true



Everything you just learned for IntList — append, get, set, size, remove, hasValue, index — works identically here. Learn the moves once, use them on both.



A few text-only bonus moves


Because the values are words, StringList throws in some helpers that wouldn't make sense for numbers:


names.sort();    // puts the words in alphabetical order
names.lower();   // makes every entry lowercase
names.upper();   // makes every entry UPPERCASE



And the loop? Same shape again — greet every name in the list:


for (int i = 0; i < names.size(); i++) {
  println("Hello, " + names.get(i) + "!");
}




  String gotcha — don't forget! ↓
  When you compare the strings you pull out of a list, use .equals(), not ==. So names.get(i).equals("Sam") — never names.get(i) == "Sam". With text, == checks the wrong thing and will burn you.

  The two mistakes everyone makes ↓
  1. Writing a.length() on an array (it's .length, no parentheses) — or a.size on a list (it's .size(), with parentheses).

  2. Trying list[i] on a list (use .get(i)) or array.get(i) on an array (use [i]).


  
The payoff

What can you actually DO with a pile of values?


Storing values is only half the story. The real power is answering questions about all of them at once: What's the total? The highest? How many are 95? Is a value even in there? Every one of these is the same loop wearing a different hat.



They all share one rhythm — the accumulator pattern. You keep a single variable holding "the answer so far," and the loop updates it one item at a time:




  
1Before the loop
Make a variable to hold the answer-in-progress.
int total = 0;

  
2During each pass
Update it using the current item.
total += scores[i];

  
3After the loop
The variable now holds your answer.
println(total);





1 · Adding it all up — and the average


Start the total at 0, then pour each value into it. Once you have the total, the average is just the total divided by how many there are.


int total = 0;                          // before: an empty bucket
for (int i = 0; i < scores.length; i++) {
  total = total + scores[i];            // during: add this one in
}
float average = (float) total / scores.length;   // after
println("Total: " + total + "   Average: " + average);


Why the (float)? ↓Without it, int / int throws away the decimals — 7 / 2 gives 3, not 3.5. Casting one side to float keeps the fraction.



2 · The biggest and the smallest


Keep a "champion" variable. Start it at the first value, then challenge it against every other value — if you find a bigger one, crown a new champion.


int highest = scores[0];                // before: assume the first is best
for (int i = 0; i < scores.length; i++) {
  if (scores[i] > highest) {            // during: found a bigger one?
    highest = scores[i];                // crown the new champion
  }
}
println("Highest: " + highest);


Smallest is the exact mirror: start int lowest = scores[0]; and flip the comparison to scores[i] < lowest.


Don't start at 0! ↓If you wrote int highest = 0; and every score happened to be negative, your loop would report 0 as the "highest" — a value that isn't even in the list. Starting at scores[0] guarantees your champion is always a real value.



3 · How many times does a value appear?


A counter. Start at 0, and add 1 every time the current item matches what you're hunting for.


int count = 0;                          // before
for (int i = 0; i < scores.length; i++) {
  if (scores[i] == 95) {                // during: is this one a match?
    count = count + 1;                  // tally it
  }
}
println("95 appears " + count + " times");



4 · Is the value in there at all?


If you only care whether something exists (not how many), use a boolean flag. Start it false, and flip it to true the moment you spot a match.


boolean found = false;                  // before: haven't seen it yet
for (int i = 0; i < scores.length; i++) {
  if (scores[i] == 95) {
    found = true;                       // during: saw it!
  }
}
println("Contains 95? " + found);



Lists hand you some of these for free ↓
An IntList has built-in shortcuts so you don't always have to write the loop: scores.sum(), scores.min(), scores.max(), scores.hasValue(95), and scores.index(95) (first position, or −1 if absent). A StringList gives you hasValue() and index() too. But notice what's missing: there's no built-in "count how many times" — for that you always write the loop. And plain arrays give you none of these. That's the whole reason the loop versions are the skill worth owning: they work on everything.



5 · Change every item (a different beast)


The four patterns above all reduce the array down to one answer — a sum, a max, a count. This one is the opposite: the loop writes back into the array, changing every element. There's no accumulator at all. Say you want to curve every grade up by 5:


for (int i = 0; i < scores.length; i++) {
  scores[i] = scores[i] + 5;   // overwrite each box with a new value
}


After the loop, the array itself has changed — every score is 5 higher than it started. The same shape works to double every value, cap everything at 100, swap the sign — anything where you're modifying the collection rather than summarizing it.

Check yourself


Predict the answer in your head first, then reveal. If you can explain the "why," you've got it.




  

    
1. For int[] colors = {7, 4, 9};, what does colors.length give — and why is it a mistake to write colors.length()?

    
    
3. The array has three boxes. On an array, length is a property with no parentheses — adding () tries to call it like a method and won't compile. (Parentheses belong to .size(), which is the list version.)

  


  

    
2. You run: IntList nums = new IntList(); nums.append(5); nums.append(9); nums.append(2);  What is nums.size()?

    
    
3. It started empty (size 0), and each append added one item to the end: [5, 9, 2].

  


  

    
3. Continuing from above, you call nums.remove(0);  Now what does nums.get(0) return?

    
    
9. Removing index 0 deletes the 5, and everything shifts left to fill the gap: the list is now [9, 2], so index 0 holds 9. (Remove doesn't leave an empty hole.)

  


  

    
4. What does this loop print?

int[] vals = {2, 8, 5, 8};
for (int i = 0; i < vals.length; i++) {
  if (vals[i] == 8) {
    println("found 8 at index " + i);
  }
}

    
    
Two lines:
found 8 at index 1
found 8 at index 3
The loop visits all four boxes; the if only prints when the value equals 8, which happens at indexes 1 and 3.

  


  

    
5. A roster of players keeps changing all game — people join and quit constantly. Array, or list? Which one, and why?

    
    
A list (here, a StringList of names). Because the number of players changes while the program runs, you need something that can append and remove. An array's locked size can't handle "keeps changing."
