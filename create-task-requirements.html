What your Create Task actually needs

Two big things. A function that takes a parameter, uses it in a conditional, and runs a loop — and that gets called. Plus a list or array that stores data you then put to work. This page breaks down each piece so you know exactly what counts.

01

The big picture

Everything on the rubric comes down to two requirements:

requirement 1 A function (procedure) that has a parameter, where that parameter is used in a conditional, and the function also contains a loop. And it has to be called somewhere.

requirement 2 A list or array that stores a collection of data, where you then use that data — finding whether something is in it, counting how often, or checking how many items meet a criteria.

make it your own Every example here shows the shape of a requirement, not your project. The program you turn in has to be your own idea, written by you. Use these to understand what is being asked, then build something that fits your sketch. I can't help you write your Create Task — but I can make sure you know the target.

03

The parameter

A parameter is an input written in the parentheses of your function. It lets the same function behave differently depending on what you pass in.

void greet(int size) {
  textSize(size);          // the parameter actually does something
  text("hi", 50, 50);
}
strict rule The parameter has to be used. A parameter that sits in the parentheses but never appears in the body is an unused parameter — that's an automatic deduction, not a style nitpick.
04

The conditional

Your parameter has to steer a conditional — an if (usually with an else). The test itself must depend on the parameter.

void checkSpeed(int speed) {
  if (speed > 10) {
    fill(255, 0, 0);       // fast -> red
  } else {
    fill(0, 0, 255);       // slow -> blue
  }
}
the test must use the parameter if (speed > 10) counts because it checks the parameter. An if (mousePressed) sitting next to your parameter does not — the condition has to read the value you passed in.
05

The loop

The same function needs a loop — a for or while that repeats something. Often the parameter decides how many times.

void drawRow(int count) {
  for (int i = 0; i < count; i++) {
    ellipse(30 + i * 40, 60, 24, 24);   // positioning math goes inside
  }
}
how we write loops in here Start your counter at 0, go up by 1, and keep the positioning math (30 + i * 40) inside the loop body. That keeps each repeat lined up.
06

Calling it

Writing a function only describes what it could do. A function that is never called never runs. You call it by writing its name with values in the parentheses.

void draw() {
  background(255);
  drawRow(6);          // call it here -> it runs every frame
}

void mousePressed() {
  drawRow(3);          // ...or call it when something happens
}
easy to forget If you define a great function and your screen stays blank, check first that you actually called it. Common spots: inside draw(), or inside an event like mousePressed() or keyPressed().
07

Your data: a list or an array

Requirement 2 wants a collection — many values held together under one name. You have two main options.

A fixed-size array

Great when you know how many items up front.

int[] scores = {88, 95, 72, 60, 91};   // a fixed-size array
println(scores[0]);                    // read one item -> 88
println(scores.length);                // how many items -> 5

A list that can grow (ArrayList)

Great when items get added while the program runs.

ArrayList<Integer> scores = new ArrayList<Integer>();
scores.add(88);          // lists can grow
scores.add(95);
println(scores.get(0));  // read one item -> 88
println(scores.size());  // how many items -> 2
remember Array length uses .length; ArrayList uses .size(). Array items use scores[0]; ArrayList items use scores.get(0). Both start counting at index 0.
08

Using the data

Storing data isn't enough on its own — you have to do something with it. The list should be pulling real weight. These three patterns are the most useful (and they all loop through the collection):

Does it exist?

// is the value in the array at all?
boolean found = false;
for (int i = 0; i < scores.length; i++) {
  if (scores[i] == target) {
    found = true;
  }
}

How many times?

// how many times does it appear?
int count = 0;
for (int i = 0; i < scores.length; i++) {
  if (scores[i] == target) {
    count = count + 1;
  }
}

How many meet a criteria?

// how many meet a criteria?
int passing = 0;
for (int i = 0; i < scores.length; i++) {
  if (scores[i] >= 70) {
    passing = passing + 1;
  }
}

09

Three array patterns worth memorizing

Almost every "use the data" requirement is one of these three moves. Learn the shape of each and you can adapt it to nearly anything. The only difference between the two columns is how you read an item and ask for the size — flip the toggle to compare.

1 — Does a value exist at all?

Start by assuming you haven't found it. Walk the whole collection; the instant you hit a match, flip the flag to true.

boolean exists(IntList array, int searchItem) {
  boolean found = false; //assume we didn't find it until we prove we did
  for (int i = 0; i < array.size(); i++) {
    if (array.get(i) == searchItem) {
      found = true;
    }
  }
  return found;
}
boolean exists(int[] array, int searchItem) {
  boolean found = false; //assume we didn't find it until we prove we did
  for (int i = 0; i < array.length; i++) {
    if (array[i] == searchItem) {
      found = true;
    }
  }
  return found;
}

2 — Find an "extreme" value (largest or smallest)

Assume the first item is the biggest, then let anything bigger take its place as you scan.

int findMax(IntList myArray) {
  int largest = myArray.get(0);
  for (int i = 0; i < myArray.size(); i++) {
    if (myArray.get(i) > largest) {
      largest = myArray.get(i);
    }
  }
  return largest;
}
int findMax(int[] myArray) {
  int largest = myArray[0];
  for (int i = 0; i < myArray.length; i++) {
    if (myArray[i] > largest) {
      largest = myArray[i];
    }
  }
  return largest;
}
smallest instead?Same pattern — rename largest to smallest and flip the > to a <.

3 — Count how many times something occurs

Start a counter at 0 and add one every time the test is true. Swap the test (== 3 → >= 70) and the same code now counts "how many meet a criteria."

int countNumberOfThrees(IntList array) {
  int count = 0;
  for (int i = 0; i < array.size(); i++) {
    if (array.get(i) == 3) {
      count++;
    }
  }
  return count;
}
int countNumberOfThrees(int[] array) {
  int count = 0;
  for (int i = 0; i < array.length; i++) {
    if (array[i] == 3) {
      count++;
    }
  }
  return count;
}
the only differencesRead an item: array.get(i) vs array[i]. Ask the size: array.size() vs array.length. Everything else — the loop, the if, the flag or counter — is identical.

Advanced building blocks (optional power-ups)

None of this is required for the Create Task — but if your idea wants the user to type something in, run a timer, keep score, or move lots of objects at once, here's how. Each card has a live widget to play with and the real Processing code underneath.

still your own workThese are tools, not a project. Borrow a technique, understand it, and rebuild it your way — don't paste a sketch in and call it yours.
inputAsk the user to type something

The JOptionPane import gives you a pop-up box. Whatever they type comes back as a String; wrap it in int() to use it as a number. Here it sets a grade cutoff and an appeals() function loops the list.

import javax.swing.JOptionPane;   // brings in the pop-up input box

IntList grades = new IntList(86, 89, 92, 94, 87, 84);

void setup() {
  String input = JOptionPane.showInputDialog("Enter the appeal grade cutoff:");
  int number = int(input);     // turn the typed text into a number
  appeals(number);             // pass it in as a parameter
}

void appeals(int threshold) {
  for (int i = 0; i < grades.size(); i++) {
    if (grades.get(i) >= threshold) {
      println("Congrats! " + grades.get(i) + " met the cutoff of " + threshold);
    } else {
      println(":( " + grades.get(i) + " didn't meet the cutoff");
    }
  }
}

whileKeep asking until they're right

A while loop repeats as long as its test stays true — perfect for "try again" until the answer is correct. Notice the counter is created before the loop and updated inside it.

What year did Columbus sail the ocean blue?

import javax.swing.JOptionPane;
int count = 1;   // they make at least one guess (the while loop might never run)

String input = JOptionPane.showInputDialog("What year did Columbus sail the ocean blue?");
int answer = int(input);

while (answer != 1492) {            // keep prompting while the answer is wrong
  input = JOptionPane.showInputDialog("Incorrect. Try again.");
  answer = int(input);
  count = count + 1;                // variables made OUTSIDE, updated INSIDE
}
println("It took you " + count + " guesses.");

timeA countdown clock

millis() is the number of milliseconds since your sketch started. Divide by 1000 for seconds, subtract from a limit, and you have a countdown. noLoop() freezes draw(); loop() wakes it back up.

Time remaining: 5
real clock (Calendar):––:––
int timeLimit = 5;        // seconds
int startTime = 0;
boolean timeUp = false;   // are we still running?

void draw() {
  background(255);
  if (timeUp == false) {
    text("Time remaining: " + timeRemaining(), 20, 30);
  } else {
    text("GAME OVER", 20, 30);
    noLoop();             // stop draw() from looping
  }
}

int timeRemaining() {
  int timeElapsed = millis() / 1000;          // millis() = ms since the sketch began
  int timeLeft = (timeLimit - timeElapsed) + startTime;
  if (timeLeft <= 0) { timeUp = true; }
  return timeLeft;
}

void keyPressed() {
  if (key == 'r') {                           // press R to restart
    loop();
    timeUp = false;
    startTime = millis() / 1000;
  }
}

Reading the real-world time

For the actual time of day, ask the computer's Calendar.

import java.util.Calendar;   // the computer's real clock

Calendar calendar = Calendar.getInstance();
int currentHour   = calendar.get(Calendar.HOUR_OF_DAY);
int currentMinute = calendar.get(Calendar.MINUTE);

int nowMinutes    = currentHour * 60 + currentMinute;
int targetMinutes = 12 * 60;                 // e.g. noon
int minutesUntil  = targetMinutes - nowMinutes;
stateA counter you can see

One variable (lives) plus a loop that draws one bar per life. Change the variable and the picture changes with it — the heart of any score, health bar, or level.

int lives = 3;

void draw() {
  background(255);
  for (int i = 0; i < lives; i = i + 1) {
    rect(i * 150 + 200, 200, 100, 400);     // one bar for every life left
  }
  if (lives == 0) {
    text("No more lives :(", width/2, height/2);
  }
}

void keyPressed() {
  if (keyCode == 32) {        // spacebar -> lose a life
    lives = lives - 1;
  }
  if (key == 'r') {           // R -> reset
    lives = 3;
  }
}

arraysA scored quiz with parallel lists

This is the big one: parallel lists. teams, stadiums, and correct all line up by index — item i in each list is about the same question. Answer all three to see your score build.

StringList teams    = new StringList("Braves", "Red Sox", "Cubs");
StringList stadiums = new StringList("truist", "fenway", "wrigley"); // stored lowercase
IntList correct = new IntList();   // a PARALLEL list: was each answer right?

for (int i = 0; i < teams.size(); i++) {
  String input = JOptionPane.showInputDialog("Stadium for: " + teams.get(i));
  if (input.toLowerCase().equals(stadiums.get(i))) {   // case-insensitive check
    correct.append(1);            // right
  } else {
    correct.append(0);            // wrong
  }
}

int total = 0;
for (int i = 0; i < correct.size(); i++) {   // add up the 1s
  total = total + correct.get(i);
}
println("You got " + total + " out of " + teams.size());

arraysLots of objects that move and react to the mouse

To manage many objects, give each one an index and keep its data in parallel lists (x, y, squareSize). One loop sets them up, another loop draws and moves all of them, and a helper function checks the mouse against any single one.

Move your mouse over the cubes — whichever one the mouse is inside turns coral. That's wasCollided(i) returning true. The small numbers are each cube's index i.

int numCubes = 30;
IntList x = new IntList();          // three PARALLEL lists -> one entry per cube
IntList y = new IntList();
IntList squareSize = new IntList();

void setup() {
  size(800, 800);
  for (int i = 0; i < numCubes; i++) {            // build every cube once
    squareSize.append(int(random(20, 50)));
    x.append(int(random(0, width - squareSize.get(i))));
    y.append(int(random(0, height - squareSize.get(i))));
  }
}

void draw() {
  background(255);
  for (int i = 0; i < numCubes; i++) {
    if (wasCollided(i) == true) { fill(255, 0, 0); } else { fill(0); }
    rect(x.get(i), y.get(i), squareSize.get(i), squareSize.get(i));
    text(i, x.get(i), y.get(i) - 5);              // label each cube with its index
    x.set(i, x.get(i) - 2);                       // slide left 2 px
    if (x.get(i) < 0 - squareSize.get(i)) {       // gone off the left? wrap to the right
      x.set(i, width + squareSize.get(i));
      y.set(i, int(random(0, height - squareSize.get(i))));
    }
  }
}

boolean wasCollided(int whichCube) {              // is the mouse inside this cube?
  if (mouseX > x.get(whichCube) &&
      mouseX < x.get(whichCube) + squareSize.get(whichCube) &&
      mouseY > y.get(whichCube) &&
      mouseY < y.get(whichCube) + squareSize.get(whichCube)) {
    return true;
  } else {
    return false;
  }
}
